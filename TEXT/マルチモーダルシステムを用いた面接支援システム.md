# AI面接練習支援システム 要件定義書

## 1. 背景・目的

就職活動、転職活動、社内選考などにおいて、面接対策は重要である。一方で、面接練習は相手の確保が難しく、回答内容に対する客観的なフィードバックを得にくいという課題がある。

本システムは、音声認識、音声合成、LLMを用いて、ユーザがAI面接官と会話形式で面接練習を行える環境を提供する。面接中は、ユーザの回答内容をもとに追加質問や深掘り質問を行い、面接終了後には回答の抽象度、矛盾点、深掘り不足、改善例などを提示する。

これにより、ユーザが実際の面接に近い形で練習し、自身の回答内容を客観的に振り返り、改善できる状態を目指す。

## 2. システム概要

本システムは、ユーザが音声でAI面接官と対話しながら面接練習を行う会話型面接支援システムである。

ユーザの音声回答は音声認識によりテキスト化され、LLMによって回答内容の分析、矛盾検出、追加質問生成、フィードバック生成が行われる。AI面接官の質問や応答は、VOICEVOXを用いて音声出力する。

面接終了後、システムは面接全体の会話履歴を分析し、回答の良かった点、抽象的だった箇所、矛盾点、深掘り不足、改善例、次回の練習方針を提示する。

## 3. 想定ユーザ

| ユーザ種別 | 説明 |
|---|---|
| 面接練習ユーザ | 就職活動、転職活動、社内面接などに向けて面接練習を行う個人利用者 |

本システムは、基本的に個人利用を想定する。初期開発では、管理者ロールを独立して設けず、面接練習ユーザ自身がアカウント作成、基本情報登録、面接練習、履歴確認、設定変更を行う。

将来的な拡張として、面接テンプレート管理、組織単位の利用状況管理、管理者向け設定機能を追加する可能性はあるが、MVPの対象ユーザには含めない。

## 4. 利用シーン

本システムは、以下のような場面での利用を想定する。

| 利用シーン | 内容 |
|---|---|
| 就職活動の面接練習 | 新卒採用面接に向けて、自己PR、学生時代に力を入れたこと、志望動機などを練習する |
| 転職活動の面接練習 | 職務経歴、転職理由、実績、キャリアプランなどを練習する |
| 特定企業向け面接練習 | 企業情報や職種情報をもとに、想定質問への回答を練習する |
| 苦手質問の反復練習 | 抽象的になりやすい質問や、矛盾が出やすい質問を重点的に練習する |

## 5. システムスコープ

### 5.1 対象範囲

本システムの対象範囲は以下とする。

| 区分 | 対象 |
|---|---|
| 初回利用時 | アカウント作成、基本情報入力、学歴・職歴の登録 |
| 面接前 | 面接テーマ選択、職種・企業・練習条件の入力、登録済み基本情報の参照 |
| 面接中 | 音声入力、音声出力、本人確認・経歴確認質問、質問生成、深掘り質問、会話履歴保持、登録情報との整合性確認 |
| 面接後 | 回答分析、矛盾検出、フィードバック提示、履歴保存 |

### 5.2 対象外範囲

初期開発では以下を対象外とする。

| 項目 | 理由 |
|---|---|
| 実企業の採用選考管理 | 本システムは面接練習支援を主目的とするため |
| 候補者と企業のマッチング | 採用プラットフォーム機能ではないため |
| 面接官による手動評価ワークフロー | 初期段階ではAIによる自動フィードバックを中心とするため |
| 表情解析・視線解析 | MVPでは音声とテキスト分析を優先するため |

## 6. 業務フロー

1. ユーザがアカウント作成を行う。認証方式はGoogle OAuth等の外部認証を想定する。
2. 初期設定として、氏名、卒業または卒業見込み学歴、学部・学科、職歴など、基本的に変動しにくい情報を入力する。
3. 登録した基本情報は、後から設定画面で変更可能とする。
4. ユーザが面接練習を開始する操作を行う。
5. ユーザが面接種別、職種、練習テーマなどを選択または入力する。
6. システムは、面接で一般的に確認される氏名、学歴、職歴などに関する質問を、定型文または生成文として用意する。
7. VOICEVOXがAI面接官として質問を音声出力する。
8. ユーザが音声で回答する。
9. 音声認識によりユーザの回答をテキスト化する。
10. LLMが、ユーザ回答と事前登録済みの基本情報に差異がないか確認する。
11. 矛盾点や不明点を含む場合、LLMが確認質問または深掘り質問を生成する。
12. システムが面接条件をもとに次の質問を生成する。
13. システムは、質問の音声出力、ユーザ音声回答、音声認識によるテキスト化を繰り返す。
14. 回答が抽象的、不十分、または矛盾を含む場合、LLMが深掘り質問を生成する。
15. システムが質問生成、回答取得、回答分析、深掘り質問生成の流れを繰り返し実施する。
16. 所定の条件に達した場合、またはユーザが終了操作を行った場合、面接を終了する。
17. LLMが面接全体の会話履歴を分析する。
18. システムがフィードバック結果を画面に提示する。
19. 必要に応じて面接履歴とフィードバック結果を保存する。

## 7. 機能要件

### 7.0 アカウント・基本情報管理機能

| 項目 | 内容 |
|---|---|
| 概要 | 個人利用者がアカウントを作成し、面接練習で参照する基本情報を登録・変更する |
| 認証 | Google OAuth等の外部認証を想定する |
| 登録項目 | 氏名、学歴、卒業または卒業見込み、学部・学科、職歴、希望職種、自己PRの素材となる情報 |
| 利用目的 | 面接冒頭の確認質問、回答内容の整合性確認、質問生成、フィードバック生成に利用する |
| 変更 | 登録済み情報は設定画面から変更可能とする |
| 備考 | MVPでは、面接練習ユーザ自身が管理者を兼ねるため、独立した管理者機能は設けない |

### 7.1 面接条件設定機能

| 項目 | 内容 |
|---|---|
| 概要 | 面接練習の前提条件を設定する |
| 入力項目 | 面接種別、職種、業界、企業名、練習テーマ、面接時間、質問数、 |
| 出力 | 面接セッション設定 |
| 備考 | MVPでは職種、練習テーマ、質問数を必須項目とする。学歴・職歴などの基本情報は、面接条件とは別に登録済みプロフィールから参照する |

### 7.2 音声入力機能

| 項目 | 内容 |
|---|---|
| 概要 | ユーザの回答音声を取得し、テキスト化する |
| 入力 | マイク音声 |
| 処理 | 音声認識エンジンによる文字起こし |
| 出力 | 回答テキスト |
| エラー対応 | 音声認識に失敗した場合、再録音またはテキスト入力に切り替え可能とする |

### 7.3 音声出力機能

| 項目 | 内容 |
|---|---|
| 概要 | AI面接官の質問や応答を音声で出力する |
| 入力 | LLMが生成した質問文・応答文 |
| 処理 | VOICEVOXによる音声合成 |
| 出力 | 音声データ |
| 備考 | 話者、話速、音量などは設定可能とすることが望ましい |

### 7.4 会話型面接進行機能

| 項目 | 内容 |
|---|---|
| 概要 | AI面接官とユーザの会話を管理し、面接を進行する |
| 処理内容 | 質問生成、回答受付、会話履歴保存、次質問生成、終了判定 |
| 終了条件 | 質問数到達、制限時間到達、ユーザによる終了操作 |
| 備考 | 実際の面接に近い自然なテンポを重視する |

### 7.5 LLM質問生成機能

| 項目 | 内容 |
|---|---|
| 概要 | 面接条件と会話履歴をもとに質問を生成する |
| 入力 | 面接条件、登録済み基本情報、事前入力情報、会話履歴 |
| 出力 | 面接質問文 |
| 質問例 | 氏名・学歴・職歴の確認、自己PR、志望動機、職務経験、成功体験、失敗経験、キャリアプラン |

### 7.6 深掘り質問生成機能

| 項目 | 内容 |
|---|---|
| 概要 | ユーザ回答に対して追加質問や深掘り質問を生成する |
| 判定観点 | 抽象性、具体性不足、根拠不足、論理の飛躍、矛盾、質問への未回答 |
| 出力 | 深掘り質問文 |
| 例 | 「具体的にどのような行動を取りましたか」「その成果を数値で説明できますか」 |

### 7.7 矛盾検出機能

| 項目 | 内容 |
|---|---|
| 概要 | 回答内容の整合性を確認し、矛盾点を検出する |
| 比較対象 | 面接中の発話同士、登録済み基本情報、事前入力情報、解析データ、過去の回答 |
| 出力 | 矛盾の可能性がある箇所、理由、確認すべき質問 |
| 備考 | LLMによる判定は断定ではなく、確認候補として扱う |

### 7.8 回答分析機能

| 項目 | 内容 |
|---|---|
| 概要 | ユーザ回答の品質を分析する |
| 分析観点 | 抽象度、具体性、論理性、一貫性、質問適合性、根拠、数値具体性、表現品質 |
| 出力 | 分析結果、スコア、コメント |
| 備考 | スコアだけでなく、該当発話と理由を提示する |

### 7.9 フィードバック生成機能

| 項目 | 内容 |
|---|---|
| 概要 | 面接終了後に総合フィードバックを生成する |
| 入力 | 面接条件、会話履歴、回答分析結果、矛盾検出結果 |
| 出力 | 総合評価、良かった点、改善点、抽象的だった箇所、矛盾点、深掘り不足、改善例 |
| 備考 | ユーザが次回練習に活かせる具体的な表現とする |

### 7.10 面接履歴管理機能

| 項目 | 内容 |
|---|---|
| 概要 | 面接練習の履歴を保存・参照する |
| 保存対象 | 面接条件、質問、回答テキスト、分析結果、フィードバック |
| 任意保存 | 音声データの保存有無は設定可能とする |
| 備考 | 個人情報保護の観点から削除機能を設ける |

## 8. AI要件

### 8.1 LLMの役割

LLMは以下の処理を担当する。

| 役割 | 内容 |
|---|---|
| 質問生成 | 面接条件に応じた質問を生成する |
| 深掘り | 回答の不足点に応じた追加質問を生成する |
| 矛盾検出 | 会話履歴や事前情報との不整合を検出する |
| 回答分析 | 抽象度、具体性、論理性などを評価する |
| フィードバック | 面接終了後の総合的な改善提案を生成する |

### 8.2 分析観点

| 観点 | 判定内容 |
|---|---|
| 抽象度 | 抽象的な表現に偏っていないか |
| 具体性 | 事実、経験、行動、成果が具体的に述べられているか |
| 数値具体性 | 成果や規模を数値で説明できているか |
| 一貫性 | 回答同士に矛盾がないか |
| 質問適合性 | 質問に対して適切に答えているか |
| 論理性 | 結論、理由、具体例の流れが自然か |
| 再現性 | 成果につながった行動が再現可能な形で説明されているか |
| 深掘り余地 | 面接官が追加で確認すべき未説明部分があるか |

### 8.3 LLM利用時の制約

| 項目 | 内容 |
|---|---|
| 根拠提示 | フィードバックには根拠となる発話箇所を可能な限り紐づける |
| 断定回避 | 矛盾検出や評価は断定ではなく、確認・改善候補として提示する |
| 個人情報保護 | 入力データ、会話履歴、音声データの取り扱いに注意する |
| 再現性 | 同一入力に対して極端に異なる評価にならないようプロンプトを管理する |

## 9. 外部連携要件

| 連携対象 | 用途 |
|---|---|
| 音声認識エンジン | ユーザ音声のテキスト化 |
| VOICEVOX | AI面接官の音声出力 |
| LLM API | 質問生成、分析、フィードバック生成 |
| データベース | 面接履歴、分析結果、設定情報の保存 |

音声認識エンジンは、Web Speech API、Whisper、クラウド音声認識サービスなどから選定する。MVPでは実装容易性と日本語認識精度を重視する。

## 12. 非機能要件

| 区分 | 要件 |
|---|---|
| 応答性能 | ユーザ回答後、数秒以内に次の質問または応答を返す |
| 可用性 | 音声入力または音声出力に失敗しても、テキスト入力・表示で継続可能とする |
| セキュリティ | 面接履歴、音声データ、個人情報を適切に保護する |
| プライバシー | 音声データの保存有無、保存期間、削除方法を明示する |
| 操作性 | 面接中に迷わず録音、回答、終了操作ができるUIとする |
| 監査性 | フィードバックの根拠となる発話を追跡できるようにする |
| 拡張性 | 職種別、業界別、企業別の面接テンプレートを追加可能とする |
| 保守性 | 音声認識、VOICEVOX、LLMの各処理を疎結合に設計する |

| 企業別面接対策 | 企業理念、求人票、過去質問をもとに質問を最適化する |
| 職種別評価基準 | エンジニア、営業、企画など職種ごとに評価観点を切り替える |

| 練習計画作成 | 過去の結果から次回練習テーマを自動提案する |
| 音声特徴分析 | 話速、間、声量などを分析する |
| 面接官キャラクター設定 | 面接官の口調、厳しさ、質問傾向を変更できるようにする |
