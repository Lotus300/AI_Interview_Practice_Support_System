# AI面接練習支援システム バックエンド詳細設計書

## 1. 目的

本書は、承認済みのバックエンド設計書をもとに、TypeScript + Fastify + Firestoreで実装するための詳細設計を定義する。

対象は、ディレクトリ構成、モジュール責務、Controller、Service、Repository、外部API Client、DTO、エラー、認証、ジョブ処理である。

## 2. 実装方針

| 項目 | 方針 |
|---|---|
| 言語 | TypeScript |
| HTTPフレームワーク | Fastify |
| バリデーション | Zod |
| DB | Firestore |
| 認証 | Google OAuth + Cookieセッション |
| 外部API | Google Cloud Speech-to-Text V2, Gemini API, VOICEVOX Engine API |
| 非同期処理 | フィードバック生成のみジョブ型 |
| 音声保存 | 保存しない |

## 4. 共通型

### 4.1 RequestContext

```ts
type RequestContext = {
  requestId: string;
  userId?: string;
  sessionCookieId?: string;
};
```

全Serviceへ渡す。ログ、認可、監査に利用する。

## 5. Controller設計

ControllerはHTTP層の責務に限定する。

| 責務 | 内容 |
|---|---|
| 入力取得 | params, query, body, multipartを取得 |
| バリデーション | Zod schemaで検証 |
| Service呼び出し | 業務処理をServiceへ委譲 |
| レスポンス返却 | DTOをHTTPレスポンスへ変換 |

ControllerでFirestoreや外部APIを直接呼ばない。

## 6. Service設計

### 6.1 AuthService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `getCurrentUser(ctx)` | `RequestContext` | `UserDto` | Cookieセッションからログインユーザを取得 |
| `handleOAuthCallback(code)` | `string` | `AuthResult` | Google OAuth結果を検証し、ユーザとセッションを作成 |
| `logout(ctx)` | `RequestContext` | `void` | セッションを失効 |

### 6.2 ProfileService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `getProfile(ctx)` | ctx | `ProfileDto` | ユーザプロフィールを取得 |
| `upsertProfile(ctx, input)` | ctx, `ProfileInput` | `ProfileUpdateResult` | プロフィール保存、`profileCompleted` 更新 |

### 6.3 SettingsService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `getSettings(ctx)` | ctx | `SettingsDto` | 設定取得。未作成なら初期値を返す |
| `updateSettings(ctx, input)` | ctx, `SettingsInput` | `SettingsDto` | 音声・モデル設定を保存 |

`saveAudio` はMVPでは必ず `false` とする。

### 6.4 InterviewSessionService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `createSession(ctx, input)` | ctx, `CreateSessionInput` | `InterviewSessionDto` | 面接条件を保存 |
| `getSession(ctx, sessionId)` | ctx, string | `InterviewSessionDetailDto` | セッション、質問、会話履歴を取得 |
| `listSessions(ctx, query)` | ctx, query | `SessionListDto` | 履歴一覧を取得 |
| `finishSession(ctx, sessionId, input)` | ctx, string, input | `FinishSessionDto` | 面接を終了 |
| `deleteSession(ctx, sessionId)` | ctx, string | `void` | 論理削除 |

### 6.5 QuestionService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `generateInitialQuestion(ctx, sessionId)` | ctx, string | `QuestionDto` | Geminiで初回質問生成 |
| `generateNextQuestion(ctx, sessionId, input)` | ctx, string, input | `QuestionDto` | reasonに応じた質問生成 |

質問生成時は、`questions` と `utterances` を追加し、`interviewSessions.currentQuestionId` を更新する。

### 6.6 AnswerService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `submitAnswer(ctx, sessionId, input)` | ctx, string, `AnswerInput` | `AnswerAnalysisResult` | 回答保存、Gemini分析、会話履歴追加 |

保存対象は回答テキストであり、音声ファイルは保存しない。

### 6.7 SpeechService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `recognize(ctx, input)` | ctx, audio input | `SpeechRecognizeResult` | Speech-to-Textで文字起こし |

失敗時は `recognition_failed` と `recoveries` を返す。

### 6.8 VoiceService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `synthesize(ctx, input)` | ctx, `VoiceSynthesizeInput` | `VoiceSynthesizeResult` | VOICEVOXで音声生成 |

VOICEVOX失敗時は `text_only` を返し、面接継続を可能にする。

### 6.9 FeedbackService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `startFeedbackJob(ctx, sessionId)` | ctx, string | `JobAcceptedDto` | feedback_generationジョブを作成 |
| `getFeedback(ctx, sessionId)` | ctx, string | `FeedbackDto` | 生成済みフィードバックを取得 |
| `generateFeedback(jobId)` | string | `void` | ジョブ内でGeminiにより生成し保存 |

### 6.10 JobService

| メソッド | 入力 | 出力 | 内容 |
|---|---|---|---|
| `getJob(ctx, jobId)` | ctx, string | `JobDto` | ジョブ状態を取得 |
| `markRunning(jobId)` | string | void | 実行中へ更新 |
| `markCompleted(jobId, resultRef)` | string, map | void | 完了へ更新 |
| `markFailed(jobId, error)` | string, error | void | 失敗へ更新 |

## 7. Repository設計

RepositoryはFirestoreアクセスのみを担当する。業務判断はServiceに置く。

### 7.1 共通ルール

| ルール | 内容 |
|---|---|
| 認可 | `userId` 条件を必ず含める |
| 論理削除 | `deletedAt == null` を基本条件にする |
| 時刻 | `createdAt`, `updatedAt` はサーバ時刻で設定 |
| トランザクション | 複数ドキュメント更新時に使用 |

### 7.2 主なRepository

| Repository | 主なメソッド |
|---|---|
| `UserRepository` | `findById`, `findByGoogleSub`, `create`, `updateProfileCompleted`, `updateLastLoginAt` |
| `AuthSessionRepository` | `create`, `findValidByHash`, `revoke`, `deleteExpired` |
| `ProfileRepository` | `findByUserId`, `upsert` |
| `SettingsRepository` | `findByUserId`, `upsert` |
| `InterviewSessionRepository` | `create`, `findByIdForUser`, `listByUser`, `updateStatus`, `finish`, `softDelete` |
| `QuestionRepository` | `create`, `listBySession`, `findById` |
| `AnswerRepository` | `create`, `listBySession`, `findById` |
| `UtteranceRepository` | `create`, `listBySessionOrdered`, `getNextSequenceNo` |
| `FeedbackRepository` | `create`, `findBySessionId`, `updateStatus` |
| `JobRepository` | `create`, `findByIdForUser`, `updateStatus`, `updateProgress` |

## 8. API処理詳細

### 8.1 `POST /interview-sessions`

```text
1. auth middlewareでuserId取得
2. request bodyを検証
3. InterviewSessionService.createSession()
4. interviewSessionsを作成
5. status=creatingで返却
```

### 8.2 `POST /speech/recognize`

```text
1. auth middlewareでuserId取得
2. multipart/form-dataを受信
3. sessionId, questionId, audioを検証
4. セッション所有者を確認
5. SpeechService.recognize()
6. Speech-to-Textへ音声を送信
7. 音声バイナリを破棄
8. transcriptを返却
```

DBには保存しない。回答確定時に `POST /answers` でテキストを保存する。

### 8.3 `POST /answers`

```text
1. auth middlewareでuserId取得
2. request bodyを検証
3. セッション状態を検証
4. answersに回答を保存
5. utterancesにuser発話を追加
6. Geminiで回答分析
7. answer.analysisを更新
8. interviewSessions.statusを更新
9. nextActionを返却
```

### 8.4 `POST /next-question`

```text
1. auth middlewareでuserId取得
2. reasonを検証
3. セッション、プロフィール、会話履歴、回答分析結果を取得
4. Geminiで次質問を生成
5. questionsに保存
6. utterancesにAI発話を追加
7. interviewSessions.currentQuestionIdを更新
8. questionを返却
```

### 8.5 `POST /voice/synthesize`

```text
1. auth middlewareでuserId取得
2. request bodyを検証
3. settingsからVOICEVOX設定を取得
4. VoicevoxClient.createAudioQuery()
5. speedScale, volumeScaleを反映
6. VoicevoxClient.synthesize()
7. WAVまたは一時再生URLを返却
```

失敗時:

```json
{
  "aiResponseStatus": "text_only",
  "voice": null
}
```

### 8.6 `POST /feedback`

```text
1. auth middlewareでuserId取得
2. セッション所有者とstatus=finishedを確認
3. jobsにfeedback_generationを作成
4. interviewSessions.feedbackStatus=generatingに更新
5. job情報とpollingUrlを返却
6. ジョブ実行処理でGeminiを呼び出す
```

MVPではAPI内でジョブ実行を開始してよい。Cloud Run上で確実性が必要な場合はCloud Tasksへ移行する。

## 10. 状態チェック詳細

| 操作 | 許可状態 | 失敗時 |
|---|---|---|
| 初回質問生成 | `creating`, `initial_question_generating` | `409 INVALID_STATE` |
| 音声認識 | `waiting_answer`, `recording`, `speech_recognizing` | `409 INVALID_STATE` |
| 回答送信 | `waiting_answer`, `answer_confirming` | `409 INVALID_STATE` |
| 次質問生成 | `answer_analyzing`, `consistency_checking`, `deep_dive_judging`, `next_question_generating` | `409 INVALID_STATE` |
| 面接終了 | `waiting_answer`, `question_presented`, `finish_confirming` | `409 INVALID_STATE` |
| フィードバック生成 | `finished` | `409 INVALID_STATE` |

## 11. トランザクション詳細

### 11.1 回答保存トランザクション

| 順序 | 処理 |
|---|---|
| 1 | `interviewSessions/{sessionId}` を取得 |
| 2 | `userId` と状態を検証 |
| 3 | `answers/{answerId}` を作成 |
| 4 | `utterances/{utteranceId}` を作成 |
| 5 | `interviewSessions.answeredCount` を更新 |
| 6 | `interviewSessions.status` を更新 |

Gemini分析結果の保存は、分析完了後に追加更新してよい。

### 11.2 フィードバック完了トランザクション

| 順序 | 処理 |
|---|---|
| 1 | `jobs/{jobId}` を取得 |
| 2 | `feedbacks/{feedbackId}` を作成 |
| 3 | `jobs.status=completed` に更新 |
| 4 | `jobs.resultRef` を更新 |
| 5 | `interviewSessions.feedbackStatus=generated` に更新 |
| 6 | `interviewSessions.summary` を更新 |

## 12. テスト観点

| 対象 | テスト観点 |
|---|---|
| AuthService | セッション有効、期限切れ、失効済み |
| ProfileService | 初回登録、更新、必須項目不足 |
| InterviewSessionService | 作成、取得、一覧、論理削除 |
| SpeechService | 認識成功、認識失敗、音声サイズ超過 |
| AnswerService | 回答保存、分析成功、状態不整合 |
| QuestionService | 通常質問、深掘り質問、確認質問 |
| VoiceService | 音声生成成功、VOICEVOX失敗時のtext_only |
| FeedbackService | ジョブ作成、生成成功、生成失敗 |
| Repository | userIdによる認可漏れ防止 |
