# AI面接練習支援システム バックエンド設計書

## 1. 目的

本書は、AI面接練習支援システムのMVPにおけるバックエンド構成、責務、モジュール、外部連携、エラー制御、デプロイ方針を定義する。

## 2. 前提

| 項目 | 方針 |
|---|---|
| 実行基盤 | Cloud Run |
| 言語 | TypeScript |
| ランタイム | Node.js |
| API形式 | REST API |
| 認証 | Google OAuth + Cookieセッション |
| DB | Firestore |
| 音声認識 | Google Cloud Speech-to-Text V2 |
| LLM | Gemini API |
| 音声出力 | VOICEVOX Engine API |
| 非同期処理 | フィードバック生成のみジョブ型 |
| 音声保存 | 保存しない |

## 3. 全体構成

```text
Browser
  -> Cloud Run Backend API
    -> Firestore
    -> Google OAuth
    -> Google Cloud Speech-to-Text V2
    -> Gemini API
    -> Cloud Run VOICEVOX Service
```

Backend APIは、画面からのすべての業務API呼び出しを受け付ける。ブラウザからGoogle Cloud API、Gemini API、VOICEVOX Engine APIを直接呼ばない。

## 4. 技術選定

### 4.1 推奨スタック

| 項目 | 採用候補 |
|---|---|
| HTTPフレームワーク | Fastify |
| バリデーション | Zod |
| OpenAPI連携 | Zod schemaから生成、またはOpenAPI定義を手動同期 |
| DB SDK | Google Cloud Firestore SDK |
| 認証 | Google OAuth client + 独自Cookieセッション |
| テスト | Vitest |
| ログ | pino |
| デプロイ | Docker + Cloud Run |

Fastifyを第一候補とする。理由は、軽量でCloud Runに載せやすく、TypeScriptとの相性がよいこと、リクエスト単位のログやバリデーションを組み込みやすいことである。

## 5. レイヤ構成

```text
src/
  app.ts
  server.ts
  config/
  routes/
  controllers/
  services/
  repositories/
  clients/
  schemas/
  jobs/
  middleware/
  utils/
```

| レイヤ | 責務 |
|---|---|
| `routes` | URL、HTTPメソッド、認証要否、バリデーションを定義 |
| `controllers` | リクエストを受け取り、サービスを呼び出し、レスポンスを返す |
| `services` | 業務ロジック、状態遷移、外部API連携の制御 |
| `repositories` | Firestoreへの読み書き |
| `clients` | Google Cloud Speech-to-Text、Gemini、VOICEVOXなど外部API呼び出し |
| `schemas` | Zodスキーマ、DTO、構造化出力スキーマ |
| `jobs` | フィードバック生成ジョブ |
| `middleware` | 認証、CSRF、エラーハンドリング、ログ |

## 6. モジュール設計

### 6.1 Auth

| 機能 | 内容 |
|---|---|
| Google OAuth開始 | GoogleログインURLを生成 |
| OAuthコールバック | Google認証結果を検証し、ユーザを作成または更新 |
| Cookieセッション発行 | `interview_session` Cookieを発行 |
| セッション検証 | `authSessions` を参照しログインユーザを特定 |
| ログアウト | セッションを失効しCookieを削除 |

CookieセッションIDは平文保存せず、ハッシュ化してFirestoreに保存する。

### 6.2 Profile

| 機能 | 内容 |
|---|---|
| プロフィール取得 | `profiles/{userId}` を取得 |
| プロフィール保存 | 学歴、職歴、自己PR素材を保存 |
| 初期登録完了更新 | `users.profileCompleted` を更新 |

### 6.3 Settings

| 機能 | 内容 |
|---|---|
| 設定取得 | 音声・モデル設定を取得 |
| 設定更新 | VOICEVOX話者、速度、音量、モデル設定を更新 |
| バリデーション | `speedScale`, `volumeScale` は0.1刻み、`saveAudio` はfalse固定 |

### 6.4 Interview Session

| 機能 | 内容 |
|---|---|
| セッション作成 | 面接条件を保存 |
| セッション取得 | 現在質問、会話履歴、進行状態を返す |
| 面接終了 | 状態を `finished` に更新 |
| 履歴一覧 | ユーザの面接履歴を取得 |
| 履歴削除 | `deletedAt` による論理削除 |

### 6.5 Speech Recognition

| 機能 | 内容 |
|---|---|
| 音声受信 | multipart/form-dataで音声を受け取る |
| 音声認識 | Google Cloud Speech-to-Text V2を呼び出す |
| 結果返却 | 文字起こしテキストと信頼度を返す |
| 失敗時 | 再録音またはテキスト入力の復旧手段を返す |

録音音声は保存しない。ログにも音声バイナリを出力しない。

### 6.6 Gemini

| 機能 | 内容 |
|---|---|
| 初回質問生成 | プロフィールに基づく定型確認質問を生成 |
| 回答分析 | 抽象度、具体性、矛盾候補、深掘り要否を構造化出力で取得 |
| 次質問生成 | 通常質問、深掘り質問、確認質問を生成 |
| フィードバック生成 | 会話履歴全体を分析してフィードバックを生成 |

Geminiの出力はJSON Schemaに従うStructured Outputsを基本とする。

### 6.7 VOICEVOX

| 機能 | 内容 |
|---|---|
| 音声クエリ生成 | VOICEVOX `/audio_query` を呼び出す |
| パラメータ反映 | `speedScale`, `volumeScale` を反映 |
| 音声合成 | VOICEVOX `/synthesis` を呼び出す |
| 音声返却 | WAVバイナリまたは一時再生URLを返す |
| 失敗時 | `text_only` として質問文のみで継続 |

MVP配置時は、Backend APIとVOICEVOXをCloud Run別サービスにすることを推奨する。

### 6.8 Feedback Job

| 機能 | 内容 |
|---|---|
| ジョブ作成 | `jobs` に `queued` のジョブを作成 |
| ジョブ実行 | Gemini APIでフィードバックを生成 |
| 結果保存 | `feedbacks` に保存 |
| 状態更新 | `jobs`, `interviewSessions` を更新 |
| ジョブ取得 | `GET /jobs/{jobId}` で進捗を返す |

MVPでは、ジョブ実行はBackend API内のワーカー処理として実装してよい。ただしCloud Runはリクエスト終了後の長時間バックグラウンド処理に注意が必要なため、確実性を優先する場合はCloud Tasksを追加する。

## 7. リクエスト処理フロー

### 7.1 面接開始

```text
1. POST /interview-sessions
2. interviewSessionsを作成
3. POST /initial-question
4. Geminiで初回質問を生成
5. questions, utterances, interviewSessionsを更新
6. POST /voice/synthesize
7. VOICEVOXで質問音声を生成
```

### 7.2 回答送信

```text
1. POST /speech/recognize
2. Speech-to-Textで文字起こし
3. 画面で文字起こしを確認
4. POST /answers
5. 回答を保存
6. Geminiで回答分析
7. nextActionを返す
```

### 7.3 次質問生成

```text
1. POST /next-question
2. reasonに応じてGeminiプロンプトを切り替える
3. Structured Outputsで質問を取得
4. questions, utterances, session状態を更新
5. 必要に応じてVOICEVOX音声生成
```

### 7.4 フィードバック生成

```text
1. POST /feedback
2. jobsにfeedback_generationを作成
3. GET /jobs/{jobId}で進捗確認
4. ジョブ内でGemini APIを呼び出す
5. feedbacksに結果を保存
6. jobsをcompletedに更新
7. GET /feedbackで結果表示
```

## 8. 状態遷移制御

Backend APIは、現在状態に応じて実行可能な操作を制限する。

| API | 実行可能な主な状態 |
|---|---|
| `POST /initial-question` | `creating`, `initial_question_generating` |
| `POST /speech/recognize` | `waiting_answer`, `recording`, `speech_recognizing` |
| `POST /answers` | `answer_confirming`, `waiting_answer` |
| `POST /next-question` | `answer_analyzing`, `consistency_checking`, `deep_dive_judging`, `next_question_generating` |
| `POST /finish` | `waiting_answer`, `question_presented`, `finish_confirming` |
| `POST /feedback` | `finished` |

不正な状態で呼び出された場合は `409 INVALID_STATE` を返す。

## 9. 外部APIクライアント

### 9.1 SpeechToTextClient

| メソッド | 内容 |
|---|---|
| `recognize(audio, options)` | 音声を文字起こしする |

入力音声は一時的にメモリまたは一時ファイルで扱い、処理後に破棄する。

### 9.2 GeminiClient

| メソッド | 内容 |
|---|---|
| `generateInitialQuestion(context)` | 初回質問を生成 |
| `analyzeAnswer(context)` | 回答分析 |
| `generateNextQuestion(context)` | 次質問生成 |
| `generateFeedback(context)` | フィードバック生成 |

各メソッドは用途別のモデル設定を参照する。

### 9.3 VoicevoxClient

| メソッド | 内容 |
|---|---|
| `createAudioQuery(text, speakerId)` | `/audio_query` 呼び出し |
| `synthesize(query, speakerId)` | `/synthesis` 呼び出し |
| `synthesizeQuestion(params)` | 自アプリ向けの音声生成処理 |

VOICEVOX接続先URLは環境変数で管理する。

| 環境変数 | 内容 |

| `DEFAULT_ANSWER_ANALYSIS_MODEL` | 既定の回答分析モデル |
| `DEFAULT_FEEDBACK_GENERATION_MODEL` | 既定のフィードバック生成モデル |

機密値はSecret Managerで管理する。

## 11. エラーハンドリング

| エラー | HTTP | code | 復旧 |
|---|---|---|---|
| 未ログイン | 401 | `UNAUTHORIZED` | ログインへ誘導 |
| 権限なし | 403 | `FORBIDDEN` | ホームへ戻す |
| 入力不正 | 400 | `VALIDATION_ERROR` | 入力修正 |
| 状態不整合 | 409 | `INVALID_STATE` | セッション再取得 |
| 音声認識失敗 | 502 | `EXTERNAL_SERVICE_ERROR` | 再録音、テキスト入力 |
| Gemini失敗 | 502 | `EXTERNAL_SERVICE_ERROR` | 再試行、固定質問で継続 |
| VOICEVOX失敗 | 502 | `EXTERNAL_SERVICE_ERROR` | 質問文のみで継続 |
| フィードバックジョブ失敗 | 500/502 | `JOB_FAILED` | 再実行 |

## 12. ログ設計

| ログ項目 | 内容 |
|---|---|
| `requestId` | リクエストID |
| `userId` | ログインユーザID |
| `sessionId` | 面接セッションID |
| `jobId` | ジョブID |
| `operation` | 実行操作 |
| `statusCode` | HTTPステータス |
| `durationMs` | 処理時間 |
| `externalService` | 呼び出した外部サービス |
| `errorCode` | エラーコード |

ログに出してはいけないもの:

| データ | 理由 |
|---|---|
| 音声バイナリ | 音声非保存方針に反する |
| CookieセッションID平文 | セキュリティリスク |
| Google OAuthシークレット | 機密情報 |
| Gemini APIキー | 機密情報 |
| Geminiへの全文プロンプト | 個人情報を含む可能性 |

## 13. セキュリティ

| 項目 | 方針 |
|---|---|
| 認証 | Cookieセッション |
| 認可 | すべてのDBアクセスで `userId` を検証 |
| CSRF | 状態変更APIでCSRF対策を行う |
| CORS | アプリのオリジンのみ許可 |
| Cookie | `HttpOnly`, `Secure`, `SameSite=Lax` |
| APIキー | Secret Manager |
| 音声 | 保存しない |

## 14. デプロイ構成

### 14.1 Cloud Runサービス

| サービス | 内容 |
|---|---|
| `interview-backend-api` | Backend API |
| `voicevox-engine` | VOICEVOX Engine API |

### 14.2 推奨配置

```text
Cloud Run: interview-backend-api
  -> Firestore
  -> Google Cloud Speech-to-Text V2
  -> Gemini API
  -> Cloud Run: voicevox-engine
```

VOICEVOXは起動時間やCPU負荷が大きくなる可能性があるため、Backend APIとは別サービスにする。

## 15. テスト方針

| テスト | 対象 |
|---|---|
| Unit Test | services, repositories, clients |
| API Test | routes, controllers |
| Integration Test | Firestore Emulator |
| External API Mock Test | Speech-to-Text, Gemini, VOICEVOX |
| Contract Test | OpenAPI JSONとの整合 |

MVPでは、外部APIを直接叩くテストと、モックによるテストを分ける。

## 16. MVP実装順序

1. Fastifyプロジェクト作成
2. 設定管理、ログ、エラーハンドリングを実装
3. Google OAuth + Cookieセッションを実装
4. Firestore repositoriesを実装
5. Profile / Settings APIを実装
6. Interview Session APIを実装
7. Speech-to-Text連携を実装
8. Gemini質問生成・回答分析を実装
9. VOICEVOX連携を実装
10. フィードバック生成ジョブを実装
11. Cloud Runデプロイ設定を作成

## 17. 後続検討事項
