# AI面接練習支援システム 実行タスク計画書

## 1. 目的

本書は、`Multimodal_system.zip` の内容と、これまで作成した基本設計・詳細設計をもとに、AI面接練習支援システムのMVP実装タスクを再構成する。

ZIP内には、静的画面モック、基本設計書、詳細設計書、既存の実行タスク計画書が含まれている。一方で、Backend APIや本番用Frontendの実装雛形はまだ作成されていない。そのため、本計画では「既存資産整理」から「実装基盤作成」「MVP縦切り実装」へ進む順序でタスクを定義する。

## 3. 実装方針

| 項目 | 方針 |
|---|---|
| 開発言語 | TypeScript |
| Backend | Node.js + Fastify |
| Frontend | TypeScript SPA |
| DB | Firestore |
| 認証 | Google OAuth + Cookieセッション |
| 音声認識 | Google Cloud Speech-to-Text V2 / Chirp 3 |
| LLM | Gemini API + Structured Outputs |
| 音声出力 | VOICEVOX Engine API |
| 音声保存 | 保存しない |
| フィードバック生成 | ジョブ型API |
| VOICEVOX話者 | 青山龍星、剣崎雌雄、No.7、東北イタコ |
| MVP対象外 | 管理者機能、複数人利用、ストリーミング音声認識 |

## 6. MVP優先実装ルート

最短で動く縦切りを作る場合は、以下の順序を優先する。

```text
T-000 -> T-001 -> T-002 -> T-004
  -> T-101 -> T-103 -> T-107
  -> T-201 -> T-205 -> T-207
  -> T-301 -> T-304 -> T-305
  -> T-501 -> T-504 -> T-505
  -> T-401 -> T-403
  -> T-405 -> T-406
  -> T-506 -> T-507 -> T-508 -> T-509
  -> T-510 -> T-604 -> T-606
  -> T-701 -> T-702 -> T-703 -> T-705 -> T-607
```

このルートでは、画面の細部や履歴詳細を後回しにし、ログイン、プロフィール、面接開始、質問生成、音声認識、音声出力、回答分析、次質問、終了、フィードバック表示までを優先する。
